什么是Make、Makefile

先来看看维基百科关于Make的解释：

In software development, Make is a command-line interface (CLI) software tool that performs actions ordered by configured dependencies as defined in a configuration file called a makefile. It is commonly used for build automation to build executable code (such as a program or library) from source code. But, not limited to building, Make can perform any operation available via the operating system shell.

概括来说Make是一种自动化执行一系列操作（比如构建程序）的命令行工具，而Makefile就是配置这些操作的文件。Make有许多不同的变种，变种之间在功能上有一些差异，这篇博客使用的是GNU Make。

Makefile的规则

Makefile由规则组成，一个规则形如：

  • target：可以是文件名（目标文件、可执行文件），也可以是一个伪目标（phony target）即一个表示动作的标签（如“clean”）
  • prerequisites：该target所依赖的一系列的文件
  • recipe：该target所需执行的一系列指令

当执行make后，make会比较target和prerequisites文件的被修改时间，若prerequisites的时间更新，recipe所定义的命令就会被执行

默认情况下，即执行make且不给任何参数：

  1. make首先会在当前目录下寻找名为Makefile或makefile的文件
  2. 如果找到，它会找到第一个目标文件（target）
  3. 如果该目标文件不存在，或是该目标文件所依赖的文件修改时间比自己新，会执行后面定义的命令来生成该目标文件
  4. 如果目标文件所依赖的文件也不存在，make又会寻找依赖文件的依赖性并生成该文件
  5. 所有依赖文件都存在，生成目标文件

可以发现整个过程中，make会一层又一层的寻找依赖关系，在找寻的过程中若发生错误（比如依赖的文件找不到）就会直接退出并报错。需要注意的是，对于定义的命令发生的错误（比如编译不成功）make并不会处理

Makefile的组成

Makefile主要包括五个东西：显式规则、隐式规则、变量定义、指令和注释

  • 显式规则：显式的说明如何生成目标文件
  • 隐式规则：make自动推导
  • 变量定义：定义的一系列变量
  • 指令：引用另一个Makefile文件；依据情况指定Makefile的有效部分（类似C语言的#if）；定义一个多行的命令
  • 注释：Makefile使用’#’表示单行注释，要想使用字符#需要使用\#

关于Makefile文件名

默认下make会在当前目录下顺序寻找名为GNUmakefile、makefile和Makefile的文件
推荐使用Makefile，因为这个文件名按按文件名顺序排序后更靠近其他重要的文件（如README）；不推荐使用GNUmakefile因为通常只有make会识别到这个文件

也可以指定Makefile的文件名，即使用-f或--file参数，如make -f Make.solaris

make在开始时会寻找include指定的其他Makefile并将他们放在对应的位置，如果在当前目录下没有找到文件，会按以下顺序继续查找：

  1. 若make使用了-I或--include-dir参数，会在给定目录继续寻找
  2. 接下来按<prefix>/include（一般是 /usr/local/bin ）、/usr/gnu/include、/usr/local/include、/usr/include的顺序寻找

若文件没有找到，make会生成一条警告信息，并继续载入其他文件，一旦完成Makefile文件的读取，make会重试之前没有找到的文件或无法读取的文件，若仍是不行则会出现一条致命警告，若想要make不管这些无法读取的文件继续执行可以使用-include或sinclude

make的工作方式

  1. 读入所有的Makefile文件
  2. 读入被include的Makefile文件
  3. 初始化文件中的变量
  4. 推导隐式规则，分析所有的规则
  5. 为所有目标文件创建依赖关系链
  6. 根据依赖关系决定那些目标需要重新生成
  7. 执行生成命令
